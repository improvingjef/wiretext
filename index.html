Sketch UI intent in text, then render ASCII or HTML instantly.

WireText helps teams and AIs communicate structure and interaction without collapsing into implementation details. Inspired by Markdown, it is readable in a terminal, friendly in docs, and fast to iterate with teammates and AIs.

How To Use It

Start minimal and keep intent visible. Use sections for structure, rows for content, and ratios only where layout must be explicit.

1
Write a surface in plain text. Use ^field, !Action, [], and simple rows.
2
Use ratio sections only when layout matters: =aside 1/6, =chat 5/6.
3
Render to ASCII for quick sanity checks and HTML for richer preview.

Syntax Cheatsheet

A compact subset carries most UI conversations.

Inputs and Actions

Design Principles

Intent over implementation

Section names are structure, not content. The notation helps teams and AIs align on what users need to see and do, not which framework to use.

Simple output contract

The renderer should be “good enough” and predictable. Preserve ratio borders and table pipes; avoid decorative complexity elsewhere.
